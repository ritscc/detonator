# Detonator — 統合ゲームデザイン文書

## 1. エグゼクティブサマリー

Detonator は、協力型ローグライト・マインスイーパーである。プレイヤーは探索・掘削・旗設置・点火（Detonate）を使ってチェックポイント（CP）を回収し、**10フロアクリア**を目指す。  
コア体験は以下の2系統の爆発システムと、時間で安全領域を削る侵食システムの同時管理にある。

- **Detonate（点火）**: プレイヤーコマンドによる管理爆発（MSTベース）。地形破壊・衝撃波なし。
- **管理外爆発（起爆）**: 危険地雷の誤掘りで発生。致死・荒地化・連鎖あり。

アイテム獲得経路は**ドロップ + レベルアップ提示のみ**。

## 3. 用語定義

### 3.1 基本用語

| 用語 | 定義 |
|---|---|
| フロア | 進行単位（1〜10） |
| ステージ | 各フロアで抽選される盤面レイアウト定義 |
| CP（チェックポイント） | セル種類ではなく「座標属性」のオブジェクト |
| frontline（前線） | 安全領域の外周に接する安全マス群（侵食起点候補） |

### 3.2 爆発系用語（統一）

| 用語 | 定義 |
|---|---|
| **Detonate（点火）** | プレイヤーコマンド。Minimum Spanning Tree(MST)で旗付き地雷を連鎖処理する管理爆発。衝撃波なし、荒地化なし。 |
| **管理外爆発（起爆）** | 危険地雷を誤って掘ったときの爆発。衝撃波・荒地化・致死を伴う。 |

> 2つは**別パイプライン**。共通点は「効果範囲内の旗とRelay Pointが除去される」点のみ。

### 3.3 セル型 enum

```ts
enum CellType {
  Safe,          // 安全マス（開放済み）
  SafeMine,      // 安全な地雷原（未開放）
  DangerousMine, // 危険な地雷原（未開放）
  Wasteland,     // 荒地
  Hole,          // 穴
}
```

## 4. セルタイプと操作可否マトリックス

| セル | 移動 | Dig | Flag | 備考 |
|---|---:|---:|---:|---|
| Safe | 可 | 不可 | 不可 | CP属性は重畳可能。Relay Pointはアイテムで設置可能。 |
| SafeMine | 不可 | 可 | 可 | DigでSafe化。Detonate連鎖上では到達時に枝停止。 |
| DangerousMine | 不可 | 可（管理外爆発） | 可 | Digで管理外爆発。 |
| Wasteland | 可（速度0.4倍） | 不可 | 不可 | 爆発/侵食で発生。 |
| Hole | 不可 | 不可 | 不可 | BridgeでSafe化可能。CP候補から除外。 |

追加ルール:

- セル変化（爆発・侵食）時、**旗とRelay Pointは即除去**。
- 侵食・管理外爆発で巻き込まれた地上ドロップは**消滅**。
- CPはセル属性と独立。侵食でセル種類が変わってもCP座標は保持される。

## 5. プレイヤー操作

### 5.1 移動・向き・衝突

- 移動は**連続座標**（グリッド吸着ではない）。
- 基礎速度: **2セル/秒**。
- 速度合成: `base × wasteland(0.4) × dash(1.5) × (1 + skillStack)`（乗算）。
- Facing（向き）: **最終移動方向固定**。
  - Dig/Flag判定: 8方向（45度刻み）
  - 地雷除去機(アイテム): 4方向（90度刻み、斜め入力は近い軸へ補正）
- プレイヤー同士はAABBコリジョンで押し合い/ブロック。物理的重なり不可。
  - 連続座標のため、理論上「同一セル内に複数人」は起こりうる。

### 5.2 Dig / Flag

- Dig/Flagの対象は地雷原セル（SafeMine / DangerousMine）のみ。
- リーチは1マス（8方向）。
- Wasteland / Hole / Safe にはDig/Flag不可。
- DangerousMineをDigすると管理外爆発。
- SafeMineをDigするとSafe化。数字0領域はflood-fillで連続開放。

### 5.3 Detonate（点火）

- 起動条件: 点火可能ノード（旗付き地雷、またはRelay Point始点）を対象にコマンド実行。
- 通常CTは式駆動（ベース10秒、人数・スキルで変動）。
- 誤フラグへの追加ペナルティCTは**なし**（連鎖停止が自然ペナルティ）。
- Force IgnitionはCTを無視するが、点火対象ノードは必要。

## 6. チェックポイントシステム

### 6.1 配置

- CPはステージ定義の候補座標から生成。
- **Holeは候補除外**。(開発者がHoleをDeterministicに決定するため、そこを候補座標から外せばよい)
- **初期安全ゾーン内は生成不可**。
- 1フロアあたりCP数は式駆動（ステージ定義 + 人数依存）。

### 6.2 可視化

- 通常: Euclidean距離で検知（`dist² ≤ R²`）、**検知者本人のみ可視**。
- Cat's eye使用時: **チーム全員に共有可視**。

### 6.3 回収と競合

- プレイヤーがCP座標に到達すると回収（掘削必須ではない）。
- 同tick同時到達はサーバー処理順で先着。

### 6.4 侵食との関係

- CPはセル属性と独立のため、侵食で消滅しない。
- 侵食由来の「CP消失」「理論上クリア不能」救済は不要。

## 7. 爆発システム

### 7.1 Detonate（点火 / 管理爆発）

### アルゴリズム

**Rooted Prim Tree + 安全地雷枝刈り**を採用。

1. 点火源から、導火線リーチ内の旗ノード群を探索（Chebyshev距離）。
2. 点火源を根にPrimでMST構築（タイブレークは決定的規則で固定）。
3. 根からBFS/キューで伝播。
   - DangerousMine到達: 爆発 + 子へ伝播継続
   - SafeMine到達: 爆発するが**その枝は停止**
   - Relay Point: Safeに設置され、中継ノードとして使用可（始点にも可）、子へ伝播継続

### 効果

- 荒地化なし
- 爆発後にSafe化
- プレイヤー致死効果なし
- 経路上の旗・Relay Pointは除去
- 連鎖速度: **1/8秒/セル**

### イベントキュー（時刻管理）

- 点火時に**タイムスタンプ付きイベント**を登録し、**3.0秒fuse後**に爆発評価。
- 爆発時点スナップショットでMST再計算（点火時表示は provisional）。
- 同tickで期限到来した点火イベントは1バッチとして取り出し、**順不同（実装依存）で段階適用**。
  - 各爆発適用後に盤面再計算し、後続イベントへ反映。
- 起爆源消失や前提崩壊（旗除去等）の保留イベントはキャンセル。
- 地雷除去機で除去された地雷に紐づくキュー済み爆発イベントもキャンセル。

### 7.2 管理外爆発（誤爆 / 誤掘り爆発）

### トリガー

- DangerousMineをDigしたとき。

### 効果プロファイル

- **衝撃波**: Chebyshev半径1（中心+周囲8マス）
- **荒地化**: Manhattan半径2（直径5相当）
- 衝撃波範囲内プレイヤーは致死（Wasteland上でも致死）
- 荒地化範囲内のSafeMine・旗・Relay Point・地上ドロップは除去/消滅

### 連鎖

- BFS連鎖（Detonateとは別系統）。
- 周囲地雷へ1/8秒遅延で伝播。

## 8. 侵食システム

### 8.1 基本挙動

- 侵食は式駆動のインターバルで発生。
- 各フェーズで侵食力（変換マス数）を算出。
- frontline候補から起点を選び、幅優先で必要マス数分だけ対象化。
- 候補不足時は深さを進める（探索深度上限なし）。

### 8.2 警告

- 警告対象は「次フェーズで変換されるマス」そのもの。
- 警告時間は式駆動（ベース3秒、侵食頻度が高いほど短縮）。

### 8.3 変換時処理

- 対象セルは地雷原（SafeMine/DangerousMine）へ変換。
- 変換時、セル上アイテムは消滅、旗/Relay Pointは除去。
- 対象セル上プレイヤーは**即死**（複数人なら同時即死）。
- 判定はtick開始位置基準。
- 各侵食フェーズ後、侵食したマスの周囲の安全マスの周囲地雷数を再計算し、数値を更新する。（新しく数字を表示するマスもあれば、数値が更新されるマスもある。）

補足:

- スポーン地点セルも侵食対象になりうる。
- Bridgeで埋めたHoleセルも侵食対象になりうる。

## 9. レベルアップとスキル

### 9.1 EXP

- SafeMineをdigするとEXP発生。
- flood-fill開放でも各マス独立でEXP発生。
- 点火コンボで倍率加算（式駆動）。

### 9.2 レベルアップフロー

- 必要EXPは式駆動（指数増加）。
- レベルアップ時に報酬を受け取る権利を獲得。
- 報酬は即選択不要で蓄積可能。
- 専用の「level-upフェーズ」は設けない。フロア間休憩時に取得することを想定するが、画面上の報酬ボタンをタップすればいつでも獲得可。
- 候補提示は有効候補のみ（満杯・上限到達で無効なものは除外）。

### 9.3 スキル仕様

- スキルはパッシブ自動適用。
- 効果量（例: +2〜6%）は取得時ランダム。
- スタック上限はスキルごとにJSONで設定（0=無制限）。
- **和音（chord）はレアスキル**として実装（1ランにつき取得は1回、取得後は常時有効）。

## 10. アイテムシステム

### 10.1 ドロップ・寿命

- SafeMine開放時に各マス独立抽選（flood-fill含む）。
- 基本ドロップ率: 10%（式補正あり）。
- 地上ドロップ寿命: **15秒**。
- 死亡者がいる場合、蘇生短縮系ドロップ比率を高める（既定90%、式補正可）。

### 10.2 所持・拾得

- ベース3枠、最大10枠（スキル拡張）。
- 枠満杯時は新規取得失敗。
- ただし同種スタック可能アイテムは、空き枠なしでもスタック加算可。
- 同種アイテムのスタック可否/上限はアイテムごとJSON定義。

### 10.3 発動方式

- 手動使用: 大半のアイテム。
- 自動発動: 九死に一生 等。

### 10.4 主要アイテム

| アイテム/名称(id) | 要点 |
|---|---|
| 中継地点(relay_point) | Safeに置くMST中継ノード。点火始点にもできる。リスポーン地点ではない。 |
| ダッシュ(dash) | 15秒、移動速度1.5倍。 |
| 強制点火(force_ignition) | CT無視でDetonate。対象ノードは必要。 |
| 地雷除去機(mine_remover)（安い/普通の/高い） | Facing準拠で前方地雷原をSafe化。 |
| キャッツアイ(cats_eye) | 最寄りCP可視化をチーム共有。 |
| 緊急脱出(evacuation) | リスポーン地点へ移動。 |
| 一息(take_a_breath)(短) / 休憩(short_break)(長) | 侵食を短時間停止。 |
| ブリッジ(bridge) | HoleをSafe化（ただし後で侵食されうる）。 |
| 使い捨て命(disposable_life) | 使用後一定時間内の死亡を回避（優先消費）。 |
| 九死に一生(nine_lives) | 死亡時に自動で死亡回避（アイテムロスト発生なし）。 |

優先消費ルール: **Disposable Life → Nine Lives**。

## 11. 死亡とリスポーン

### 11.1 死亡条件

- 管理外爆発の致死範囲に入る。
- 侵食変換セル上にいる。
- （将来イベント実装時）イベント定義で致死とされた効果を受ける。

### 11.2 死亡回避

- Disposable Life / Nine Lives は「死を回避」する。
- 回避時は死亡扱いにならず、アイテム全ロストは発生しない。

### 11.3 死亡時ペナルティ

- 所持アイテム全ロスト（死亡回避発動時を除く）。
- リスポーンタイマー開始（ベース40秒、式補正）。

### 11.4 ゴースト状態

- マップ上を移動して観戦可能。
- Ping/マーキング等の情報支援は不可。

### 11.5 リスポーン位置

- ランダムな生存プレイヤー周囲の非地雷マスへ配置。
- 候補が危険なら近傍安全マスを再探索。
- 近傍に安全マスがない場合は荒地リスポーンを許容。

### 11.6 全滅

- 全員同時死亡で即ゲームオーバー（救済なし）。
- 蘇生短縮アイテムは死亡中全員へ適用。

## 12. フロア構造と進行

### 12.1 進行単位

- 勝利は10フロアクリアのみ。
- Floor 10クリア後のエンドレスはMVP外。

### 12.2 フロアとステージ

- フロアとステージは別概念。
- 各フロアで、対応ステージプールから1つを抽選。
- 生成順序は **ステージベース値 → 人数補正**。

### 12.3 初期配置

- 初期安全ゾーンは5x5（式で変動可能）。
- 初期安全ゾーンには地雷なし、CP生成なし。
- 複数スポーン群分け（例: 2+3）を採用。

### 12.4 クリア遷移

1. 全CP回収で即時クリア
2. タイマー停止
3. 地雷原消滅（安全化）
4. 保留イベント（fuse中点火・侵食など）キャンセル
5. 全員復活 + 初期スポーン位置へ移動
6. 休憩フェーズ（報酬選択）
7. 次フロア開始

### 12.5 持ち越し/リセット

- 持ち越し: スキル、アイテム
- リセット: フラグ、地形、プレイヤー位置、CT/バフ/一時効果

### 12.6 スコア

- 基本軸: **経験値 × タイムボーナス係数**（例: `600秒(ステージごとに決まる) / クリアタイム`）
- 最小値境界あり、四捨五入。
- 同率はクリアタイムが短い方を上位。

## 14. 数式駆動設計

設計原則:

1. 主要挙動は式で定義。
2. 係数・境界条件（min/max/clamp）はJSON管理。
3. ステージ・人数・フロア進行で同一式を再利用。

主要対象パラメータ:

- 盤面サイズ（人数連動）
- SafeMine:DangerousMine比率
- 侵食インターバル
- 侵食力（変換マス数）
- 侵食警告時間
- レベルアップ必要EXP
- リスポーン時間
- Detonate CT
- ドロップ率
- CP数
- 検知半径
- コンボ倍率
- スコア計算

## 16. UI/UX設計方針

- 盤面優先・最小HUD。
- スマホ操作を最優先にUI密度を調整。
- プレイヤー表示は向き・識別色・名前を明確化。
- 侵食警告は対象マスを直接示す。
- 点火経路は「provisional（予測）」表示を許容。
- CP可視化は通常本人のみ、Cat's eye時はチーム共有。

---

## 17. マルチプレイ仕様

- チームは単一（全員協力）。
- プレイヤー当たり判定あり（押し合い/ブロック）。
- 同tick内の入力処理順は順不同（実装依存、テスト時はseed固定）。
- チャット/コミュニケーション機能はMVPで実装しない。
- 切断時は一定時間復帰待ち、タイムアウトで退出。
- 再接続時は状態完全復元（位置は危険なら近傍安全へ補正）。
- 途中参加は許可。キャッチアップなし（Lv1・アイテムなし・スキルなしで参加）。
- クライアント予測は採用せず、サーバー権威のみ。
