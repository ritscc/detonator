## 6. 矛盾レポート ⚠️

| # | 問題 | 仕様 (shared-dev-plan) | 実装 | 重大度 |
|---|---|---|---|---|
| SD-1 | ヘーラー API スタイル | coord-based (`GridCoord`) | index-based (`number`) | 中 — テストは実装に従うべき、ドキュメント要更新 |
| SD-2 | setCellFlags | `erosionWarning` を含む | flagged + hasRelayPoint のみ | 低 — 仕様は希望的目標の可能性 |
| SD-3 | createCheckpointState | `{cpId, coord}` オブジェクト | `(cpId, x, y)` 引数 | 低 — シグネチャドリフト |
| SD-4 | upsertPlayerState | `(gameState, player)` | `(playersMap, sessionId)` | 中 — 契約が異なる |
| SD-5 | resetStringArray | `(target, values)` | `(arr)` — クリアのみ | 中 — 挙動が異なる |
| SD-6 | clearAllErosionWarnings | `(grid, erosionState)` | `(grid)` のみ | **高** — warningCellKeys が陳腐したまま |
| SD-7 | resetPlayersForNewFloor | `(players, spawnBySessionId)` | `(players)` のみ | **高** — スポーンマップを無視 |
| SD-8 | 数値型 | `@type("number")` | 複フィールドが `@type("uint8")` | 低 — 最適化だがドキュメントは古い |
| **SD-9** | **フロア遷移時の EXP リセット** | **api.md は累積 EXP を示唆** | **コード+テストが exp=0 にリセット** | **📋 高 — 人間の判断が必要** |
